Difference Between session storage and local storage

SL Session Storage Local Storage
1 Session Storage data is only available to the site that created it. The stored data is not deleted when the browser is closed.
2 Session Storage data is not shared with other sites. Local storage is useful for storing data that the user will need to access later, such as offline data
3 Session Storage data is not persistent, meaning it is only available for the duration of the user's session on a site. This is especially useful for storing data that you want to persist even if the user closes the browser, such as preferences or settings.
4 It allows 5MB of data to be stored. It allows 10MB of data to be stored.

What is an event loop in JavaScript ?

Event loop is just a guardian who keeps a good communication with Call Stack and Callback Queue. It checks if the call stack is free, then lets know the callback queue. Then Callback queue passes the callback function to Call stack to be executed. When all the callback functions are executed, the call stack is out and global execution context is free.
